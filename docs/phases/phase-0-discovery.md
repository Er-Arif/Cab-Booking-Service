# Phase 0 Discovery and Assumptions

## Inputs
- Business blueprint in `madhupur_ride_booking_app_blueprint.md`
- Requirement to implement full MVP and document all phases

## Locked assumptions
- Full MVP across backend, customer app, driver app, and admin web
- Separate Flutter apps for customer and driver
- Next.js for admin panel
- Mockable external providers
- Cloud-ready local structure with Docker and env config

## Risks noted early
- Flutter SDK missing from current execution environment
- No live vendor credentials for OTP, maps, storage, or push
- No existing database migration framework yet
