# Payout and Revenue SOP

Last updated: April 16, 2026

## Goal
Track driver earnings, platform commission, and settlement notes accurately during the Version 1 pilot.

## Version 1 boundary
- payout tracking is operational, not gateway-driven
- payment status comes from ride lifecycle and operator-confirmed settlement records
- commission baseline is 15 percent unless the approved pilot config changes

## Daily review
- review completed rides
- confirm payment-complete status
- review driver earnings summaries
- review any ride with missing payment completion

## Weekly settlement routine
1. Pull completed rides for the settlement period.
2. Verify payment-complete status for each ride.
3. Compare driver earnings totals against the admin and driver-facing summaries.
4. Record payout notes for each driver internally.
5. Escalate mismatches before issuing any settlement confirmation.

## Mismatch handling
- if ride completion exists without payment completion, resolve payment status first
- if earnings totals disagree across views, stop settlement and verify the ride records
- if a complaint affects fare or payment, resolve the complaint before final settlement
