# Fare Management SOP

Last updated: April 16, 2026

## Goal
Keep Version 1 fares controlled, consistent, and auditable during the pilot.

## Baseline
- Use [version1-pilot-configuration.md](/e:/MyProjects/Cab%20Booking%20Service/docs/guides/version1-pilot-configuration.md) as the approved starting point.

## Change policy
- Do not change fares casually during active pilot observation windows.
- Any fare change should be justified by ride demand, cancellation trends, driver feedback, or support findings.
- Record the reason for every change in the pilot log.

## Update flow
1. Open the Settings page in the admin panel.
2. Review current values for bike and e-rickshaw.
3. Compare them with the approved baseline.
4. Apply only the intended fare change.
5. Save settings.
6. Confirm the updated values appear in the admin panel and new fare estimates reflect the change.

## Safe-change rule
- prefer one pricing change at a time
- observe effect before making another adjustment
- avoid changing both categories and all rate fields at once unless the pilot owner approves it
