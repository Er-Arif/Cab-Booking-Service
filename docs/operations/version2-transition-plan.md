# Version 2 Transition Plan

Last updated: April 16, 2026

## Goal
Move from a controlled Version 1 pilot to a more production-capable Version 2 release.

## First upgrades
1. Real OTP and SMS provider
2. Real maps, geocoding, and routing provider
3. Real payment and UPI gateway
4. Real push notification delivery
5. Real file and document storage

## Recommended technical follow-up
- strengthen realtime event delivery beyond polling-focused mobile sync
- expand automated integration and end-to-end test coverage
- introduce finer-grained admin permissions
- add more detailed monitoring and alerting around live provider failures

## Recommended product follow-up
- validate pilot pricing and service zones against real usage
- refine cancellation, payout, and complaint policies from pilot evidence
- formalize broader onboarding and support procedures for scale

## Release rule
Do not declare Version 2 complete until the real provider integrations and legal review track are both closed.
