# Admin Panel Usage Guide

Last updated: April 16, 2026

## Login
- open the admin panel
- sign in with the approved seeded or operator credential flow
- review the legal pack if onboarding a new operator

## Dashboard
Use the Dashboard to:
- review ride volume
- review active operations signals
- confirm the system is reflecting live backend data

## Drivers
Use Drivers to:
- review approval status
- change driver approval state
- inspect driver list and readiness for the pilot

## Customers
Use Customers to:
- inspect registered users
- review basic ride participation context for support

## Rides
Use Rides to:
- monitor current and completed rides
- verify ride lifecycle transitions during support review

## Complaints
Use Complaints to:
- investigate submitted complaints
- record resolution notes
- close resolved issues

## Reports
Use Reports to:
- review summary performance indicators
- compare ride and complaint trends over time

## Settings
Use Settings to:
- review fare defaults
- update fare values when approved through the fare SOP

## Operator discipline
- avoid changing multiple operational settings at once
- document each meaningful admin action in the pilot log
- escalate unclear cases instead of improvising risky fixes
